Aktuelle Änderungen beim Schreiben des Netzwerkes:
 * Klasse Menu
  + neue Untermenüs hinzugefügt
  + neue Attribute in Bezug auf das Netzwerk
 * Paket Input
  + Input nun ein Interface
  + Keyboard die Tastatureingabe (Methoden vereinfacht, Klasse besteht zu 80% aus Kommentaren und der Liste der Tastennamen)
  + Connector erbt auch von Input, so wird ein Spieler beim LAN gesteuert
 * Klasse Coregame
  + Es gibt eine neue Instanzvariable vom Typ Connector, diese ist null, falls es sich um kein Netzwerkspiel handelt
  + neue Spielmodi für Netzwerk
 * Klasse Bomberman
  + Spielgeschwindigeit wird besser konstant gehalten
  + Gameinstanz nun öffentlich verfügbar
 * Paket Network
  + Connection Interface, erbt von Input, um Spieler steuern zu können
  + Connector abstrakte Klasse
  + Client und Server erben von Connector
  + noch offen:
   - allgemeine Synchronisation (Karte bei Spielstart, Items, Spielerposition korrigieren)
   - Spiele auf mehreren Rechnern nur mit Änderungen des Quelltextes möglich
   		(ersetze dazu in network.Client Z. 37 "localhost" durch IP-Adresse des Servers) 
 
 Kommunikationsprotokoll
  * Spieler 1: Startet Server (ServerSocket)
  * Spieler 2: Startet Client, findet Server (Socket verbindet sich, 'Hello' wird gesendet)
  * Spieler 2: Gibt an, bereit zu sein (Sendet Information, bei Spieler 1 ändert sich eine Meldung)
  * Spieler 1: Aktiviert Spielstart
  * Server sendet die Karte (dieser Punkt ist zurzeit noch nicht implementiert)
  * Server gibt das Signal zum Spielstart
  * Spiel
   + Server und Client senden und empfangen gleichzeitig
   + beide senden Handlungen ihres Spielers
   + auf der anderen Seite wird der zweite Spieler mit diesen Informationen gefüttert
